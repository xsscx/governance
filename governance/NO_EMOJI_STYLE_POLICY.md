# No Emoji Style Policy
**Document ID:** GOV-007  
**Created:** 2026-02-07  
**Status:** Active  
**Category:** Style Preferences  
**Severity:** Style (Not a violation)

## Purpose
Document user preference for emoji-free content generation across all outputs, including source code, documentation, commit messages, and reports.

## Policy Statement
All content generated by Copilot CLI for this project shall be **emoji-free**. This applies to:
- Source code files (.cpp, .h, .c, .sh, .py, etc.)
- Documentation files (.md, .txt, .rst, etc.)
- Git commit messages
- Reports and summaries
- Comments and annotations
- Terminal output and logs
- Governance documentation

## Rationale
**User Request:** "discontinue the use of emojis in all source, documentation, commits and other content generated"

**Style Marker:** Professional, technical-only communication style preferred for:
1. **Consistency** - Uniform appearance across all content
2. **Readability** - No visual distractions in technical text
3. **Compatibility** - Universal character support across terminals/editors
4. **Professionalism** - Technical documentation standards
5. **Accessibility** - Screen readers and text-only environments

## Scope

### IN SCOPE (No Emojis)
- All `.cpp`, `.h`, `.c` source files
- All `.sh`, `.py`, `.js` scripts
- All `.md`, `.txt`, `.rst` documentation
- Git commit messages and PR descriptions
- Session reports and summaries
- Terminal output from scripts
- Error messages and warnings
- Code comments
- Governance and lesson documentation
- README files and changelogs

### OUT OF SCOPE (May Contain Emojis)
- Third-party dependencies (external libraries)
- User-provided content (issues, PRs from external contributors)
- Legacy content (historical commits, existing documentation)
- External URLs and references

## Verification Protocol

### Pre-Commit Checks
Before committing any content, verify emoji-free status:

```bash

git diff --cached | grep -E '[\u{1F300}-\u{1F9FF}]' && echo "EMOJI DETECTED" || echo "Clean"
```
